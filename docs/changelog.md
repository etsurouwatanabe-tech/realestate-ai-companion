## v1.1.2 — 2026-05-03

### 🐛 重要な不具合修正：通常市区町村の誤除外

**問題**：v1.1.0/v1.1.1 の `code[-1] == "0"` 判定が雑すぎて、末尾0の通常市区町村まで除外していた。

具体的には：
- 目黒区（13110）→ 末尾0で除外されていた
- 練馬区（13120）→ 末尾0で除外されていた
- 小金井市（13210）、東大和市（13220）等の多摩地域も同様

**修正**：政令市親判定を **「自身が『区』で終わらない」かつ「+1〜+19 範囲に区が2つ以上ある」** に変更。

これにより：
- 13100『特別区部』→ 配下に23区が並ぶ → 親判定 → 除外
- 14100『横浜市』→ 配下に18区が並ぶ → 親判定 → 除外
- 13110『目黒区』→ 自身が区 → **保持** ✓
- 13120『練馬区』→ 自身が区 → **保持** ✓
- 13210『小金井市』→ 自身が区でない、配下にも区なし → **保持** ✓

**集計対象の変化**：
- 東京：47件 → **51件**（目黒区・練馬区など4件追加）
- 神奈川：50件 → **52件**

### 謝辞
オーナー実機検証で発見・修正方針を提案いただいたバグ。検証ループの威力。

## v1.1.1 — 2026-05-03

### 🐛 マルチバージョン共存時の不具合修正

- **slash command の SKILL_DIR 取得を「最新版を自動選択」に変更**
  - 旧：`companion/*/skills/...` のワイルドカード展開（複数版あると古い方が選ばれる）
  - 新：`ls $COMPANION_DIR | sort -V | tail -1` で最新版を確実に選ぶ
- 古い cache のクリーンアップ手順を `/companion:prepare` に追加
- N03 ハンズオン記事に「アップデート後に古いエラーが出る」「正しい update コマンド」のQ&A追加

### 背景
- v1.0.x → v1.1.0 アップデート後、`~/.claude/plugins/cache/.../companion/` に
  両バージョンが並んで残り、ワイルドカード展開で v1.0.2 の prepare.py が走るケースが発生
- `--prefecture` 未対応版の prepare.py がエラーで停止する事象を実機で確認 → 修正

## v1.1.0 — 2026-05-03

### 🌏 全国対応＋slash command 化

- **prepare.py 全国対応**：`--prefecture` で都道府県名／コード／`all` 指定可
  - 単独：`--prefecture 13` / `--prefecture 東京`
  - 複数：`--prefecture 13,14,12,11`
  - 全国：`--prefecture all`
- **score.py 全国検索**：47都道府県の市区町村を統合検索
  - 末尾単位抽出：「横浜市港北区」→「港北区」で正しくマッチ
  - 同名解決：「中央区」のような全国複数候補は警告＋曖昧解決ガイド
  - 「東京:中央区」形式の都道府県明示指定をサポート
- **Slash command 2本追加**：
  - `/companion:prepare [都道府県]` — データ準備
  - `/companion:inheritance [市区町村]` — スコア確認
- **PEP 723 inline metadata**：`uv run scripts/prepare.py` で依存解決込みで完結
- **政令市親エンティティ除外**：「横浜市」「川崎市」など親市が候補に混入する不具合を解消（JIS X 0402 末尾0判定）

### 設計判断
- v1.0.x は東京23区限定で「結果の見せ直し」に寄っていた（顧問7人レビューでも指摘）
- v1.1.0 で全国対応＋対話起動を実現し、Skill 本来の本質的価値（対話で動的にカスタマイズ）に到達

### 使い方の変更
旧：`python scripts/score.py --wards "..."`
新：`/companion:inheritance 区市町村名` または `uv run scripts/score.py --wards "..."`

### 既知の制約
- 令和5年（2023年）住宅・土地統計の市区町村×年齢×所有関係データは未公表のため、世帯データは平成30年ベース
- 一部の小規模町村は e-Stat 集計対象外で取得できない場合あり
